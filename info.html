Introduction

HTML and CSS

In order to create a website, it is necessary to write code in languages, that are specifically designed to create and design websites. HTML should be thought of as the structure of the website, since it is a standard form of markup language for creating websites (HTML Tutorial, 2025). This is what allows us to create the actual content presented on the webpage, whether this be thorugh creating new headings, paragraphs or presentation of pictures, or more complex features such as for example the integration of buttons, searchbars, or other interactive user feautures (HTML Tutorial, 2025). CSS is instead the language that is used in order to style the website (CSS Tutorial, 2025). It therefore allows us to create websites that are aesthetically pleasing and engaging to interact with. For example can we make a change in color templates, texts font families and sizes, or create complex gradients and shapes (CSS Tutorial, 2025). Both HMTL and CSS should therefore be present as components when creating a website, as it has been done on this website also.




The implementation of an API

The abbrivation API stands for "Application Programming Interface" (What is an API?, 2025). APIs are sets of automated protocols that are able to allow "[...] different software components to communicate and transfer data" (What is an API?, 2025). This can be a very effective feature on a website, since it allows through small chunks of code for different pieces of software to work together in order to for example enhance user experience and business efficiency, but also for things such as enhancing online security on an app or a website. APIs have been around for centuries, and we interact with them everyday on websites on the internet; they are used on sales platforms, social media platforms, on cloud services, on mobile applications and also on devices which are able to connect to objects, such as for example cameras and microphones (What is an API?, 2025). The way they work is by data sharing through request and respond cycles - when a user interacts with an API client by for example pressing a button or searching a sepcific term on a website, a request is sent to obtain the specific piece of data. Then the API server retrieves the needed data from its ressource and this data is then returned to the user back on the website that was originally interacted with (What is an API?, 2025).

Github and its role in creating this website

Github is an open-source online environment platform, which functions as a space where users can store and share code in order to help others create code themselves, but also so that others may inspect your shared code, and post questions regarding the code, in order to help you troubleshoot and repair code that is not working optimally or at all, which promotes collaboration between users (Beshero-Bondar & Parker, 2017). Git itself is a version control system (Beshero-Bondar & Parker, 2017) and Github works as a social tool build on this version control system. What is meant by "version control system" is that it allows its users to manage changes to files and does so by storing versions of all the different varieties, that the file has existed as present and previously. This means, that the user is also able to restore previous versions of the files and see who made the changes (Beshero-Bondar & Parker, 2017). Another important feature is Githubs ability to create branches. This allows users to work on the same project, without altering or potentially breaking the main project. The projects stored on Github are restored in what is called repositories (Beshero-Bondar & Parker, 2017). These are remote folders that work as the users personal portfolio for projects and track all the changes made to the users code.
In the case of this website, I used Github to deploy my website (HMTL and CSS) as part of a repository, which made this website available - this is made possible through Github Pages site (Beshero-Bondar & Parker, 2017).
